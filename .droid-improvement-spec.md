## Required changes

### 1. BUG: `-f` is wrong for context files (HIGH PRIORITY)

Current code:
```rust
for file in &opts.context_files {
    cmd.args(["-f", file]);
}
```

Per droid help: `-f, --file <path>  Read prompt from file`. So `-f` REPLACES the prompt source, not "add a context file". Multiple `-f` is undefined and the prompt arg becomes ambiguous.

Fix: use `--append-system-prompt-file <path>` for each context file. That flag is purpose-built for this and supports being repeated.

Update tests accordingly.

### 2. Wire `opts.session_id` to `-s` (NEW)

If `opts.session_id` is `Some(id)`, pass `-s <id>`. Per droid help, `-s` requires a prompt ‚Äî we always pass one, so safe.

Mirror `src/agent/opencode.rs:53-55`.

Add a test.

### 3. Use `--use-spec` for read-only mode (BEHAVIOR FIX)

Current: `read_only` ‚Üí `--auto low`. But `--auto low` per droid docs still allows file creation/modification in non-system directories. That's NOT read-only.

Fix: when `opts.read_only` is true, pass `--use-spec` (no `--auto` flag) ‚Äî droid's spec mode is the true read-only mode per the help text ("Start in spec mode" + the default-mode description: "Read-only mode - The safest mode for reviewing planned changes without execution"). When NOT read_only, keep `--auto high`.

Update the existing read-only test to assert `--use-spec` is present and `--auto low` is NOT.

### 4. Refresh model name map (FRESHNESS)

`map_model_name`:
- `opus` ‚Üí currently `claude-opus-4-6`. Update to `claude-opus-4-7` (droid's own default per `--help`).
- Keep haiku/sonnet entries as-is (verify against the real droid model list if quick ‚Äî `droid exec --list-tools` may help, otherwise leave).
- Drop the `gpt-4.1-nano`/`gpt-4.1-mini` mappings unless we have evidence they're still meaningful ‚Äî these look like ports of stale aliases. **Don't add new ones speculatively.** If unsure, just bump opus and leave the rest.

### 5. `parse_completion` should aggregate from streamed events (LOW PRIORITY)

Current: returns `Done` with all-None. Other agents (codex, opencode) accumulate tokens/cost from the streamed `usage`/`turn_complete` events.

Implementation note: `parse_completion` only sees the final stdout buffer, not the streamed events. The simplest fix is to scan `_output` for the LAST `turn_complete`/`usage` JSON line and pull tokens/cost from it. Look at `src/agent/codex.rs::parse_completion` for the pattern.

If this turns out to require store-side state (not parser-side), defer it and just leave a `// TODO(droid): aggregate completion stats from streamed usage events` comment.
