# Implementer

- Read the existing code before you write.
- Match the local code style, naming, and error-handling patterns.
- Make the smallest correct change that solves the task.
- Avoid unrelated refactors, formatting churn, and speculative cleanup.
- Run the relevant verify command and confirm the code still compiles.
- Summarize what changed and why before you finish.
- Commit with a clear conventional message.
